一次 Waline 评论区不显示的排查记录

1. 现象回顾

  • 页面上提交评论后，当场能看到自己的评论。
  • 一刷新页面，评论区只剩输入框和「评论」两个字，历史评论全部消失。

4. 初步判断：前端内部渲染错误

综合前面的信息：

  • 配置没关评论；
  • 服务端版本正常、数据库有数据、接口返回 200 且带 9 条评论；
  • 前端控制台报的是 Waline 内部组件 WalineComment.vue 的展开语法错误。

比较合理的判断是：

服务端「给了」数据，但是 Waline 前端在解析 / 渲染时，把一个错误的数据结构做了 ... 展开，结果抛异常，导致后续 UI 不渲染。

也就是典型的 Waline 前端 bug 或服务端 / 客户端版本组合问题，而不是简单的路径配置错误。

5. 和 AI 一起的几轮试错

接下来就是我和大模型一起折腾的几轮调试，这里按时间线总结一下。

5.1 尝试统一到 Waline v2 客户端

起初怀疑是「前端 v3 + 服务端 v1.33.0」导致的兼容问题，于是先试了一版：

  1. 把 static_prefix.waline 强制成 v2：

  2. 在主题的 waline.ejs 里显式指定 v2 的多个 CDN 作为 fallback。

这一步主要是排除「前端无意中拉到 v3」的情况。
确认真的是 v2 客户端以后，WalineComment.vue:194 的错误依然存在，说明问题不只是「v3 对旧服务端不兼容」那么简单。

5.2 尝试降级到 v1 客户端（失败）

后来一度尝试让前端也降到 v1，和服务端 1.33.0 对齐：

  • 把 static_prefix.waline 指到了 v1；
  • waline.ejs 里手写 v1 的 CDN 地址。

结果发现 v1 的文件结构和现在主题期望的不完全一样，很多路径 404，直接触发我自己写的「Waline script load failed」日志，评论区连组件都挂不上，这条路线就放弃了。

  3. 部署完成后重新访问评论接口，响应头里的版本变成：

说明服务端已经升级到了较新的 1.x。

但遗憾的是，升级后前端的 WalineComment.vue:194 错误依然存在，说明问题不是简单的旧版本 bug。

5.4 完全切换到 Waline v3 客户端

参考官方文档，Waline v3 推荐的使用方式是 ESM 动态导入：

于是我和 AI 一起把 Fluid 主题内置的 Waline 集成逻辑整体重写成 v3 风格：

  • static_prefix.waline 改成 v3：

  • waline.ejs 中：

    • 用 Fluid.utils.createCssLink 加载 v3 的 waline.css；
    • 用原生 import('https://unpkg.com/@waline/client@v3/dist/waline.js') 动态加载 JS 模块；
    • 从模块中抽出 init 方法，按 v3 的方式调用；
    • 将 _config.fluid.yml 里的 serverURL/path/lang/emoji 等配置作为参数传给 init；
    • 保留主题原有的图片灯箱、图床等增强逻辑。

这一轮之后，前端的「扩展不可迭代对象」那条错误消失了，说明 v3 客户端本身是健康的。

6. 经验总结

这次踩坑主要有两个教训：

  1. 不要在 serverURL 里带上 /api
    Waline v2/v3 客户端会自己在内部拼接 /api/comment、/api/recent 等路径，serverURL 应该只写到服务端根地址即可，例如：

    如果写成 https://comment.example.com/api，就会变成 /api/api/comment，直接 404。

  2. 前后端版本要么一起升级，要么按官方推荐组合来用

    • 旧的 1.x 服务端搭配 2.x 客户端，在某些场景下会触发内部数据结构不匹配的问题；
    • 比较稳妥的做法是：服务端跟随 @waline/vercel 最新 1.x，客户端用官方文档推荐的 v3 ESM 用法。

另外，大模型在这个过程中帮了不少忙：

  • 帮我逐步排除了「配置关掉」、「接口没通」、「数据丢失」这些干扰项；
  • 给出了几种版本搭配的思路（前端 v1/v2/v3 和服务端 1.x 的不同组合）；
  • 最后协助我把 Fluid 主题内置的 Waline 集成重写成 v3 ESM 风格。

如果你也遇到了「Waline 评论提交后一刷新就不见了」或者前端报 WalineComment.vue 相关的 spread 语法错误，可以从以下几个点开始排查：

  1. 先确认接口 /api/comment 是否 200，且 errno: 0、data.count 正常；
  2. 检查 serverURL 是否多写了 /api；
  3. 查看响应头里的 x-waline-version，确认服务端版本；
  4. 根据版本选择合适的客户端接入方式（推荐 v3 ESM）。

希望这篇记录能帮你少踩几个坑。***
