博客网站的建立

这篇文章记录我从零搭起个人博客的完整过程，工具、坑点和我做出的选择。

为什么选 Hexo + Fluid

我想要一个写作体验简单、生成快、能自由掌控的方案。Hexo 只要 Node 环境就能跑，Fluid 主题颜值和可定制性都不错，于是确定组合：Hexo + Fluid。

我先装 Node.js 18，然后装 Hexo CLI。

在 ./_config.yml 把 theme 改成 fluid。我另外新建了 ./_config.fluid.yml 专门放个性化配置，后续更新主题不会丢自定义。

第四步：把站点配置对齐

在 ./_config.yml 里我设了：

  • language: zh-CN、timezone: Asia/Shanghai
  • 永久链接：permalink: :year/:month/:day/:title/
  • post_asset_folder: true（文章有自己的资源文件夹）

第五步：先美化再写

在 ./_config.fluid.yml 我做了这几件事：

  • 导航标题改成 Blog
  • 开启本地搜索、暗色模式、目录 TOC、字数统计
  • 首页和关于页加了自己的 Banner 与头像

第七步：评论系统的选择与踩坑

我先用 Waline，原因是有社交登录且可自托管。服务端我放在了 Vercel，环境变量这样配：

  • 基本：LEAN_ID、LEAN_KEY、LEAN_SERVER=https://api.lc-cn-n1-shared.com
  • 数据库：MONGO_DB、MONGO_USER、MONGO_PASSWORD、MONGO_HOST、MONGO_PORT=27017、MONGO_OPT_SSL=true
  • 安全：SECURE_DOMAINS 必须是纯域名逗号分隔（比如 shaoyanzhe.site,waline域名,localhost:4000）
  • SERVER_URL 写服务端的完整 https 地址

遇到的坑：Twitter 需要 OAuth 1.0a 的 API Key/Secret，不是 OAuth2 的 Client；SECURE_DOMAINS 写成带协议会被拒；Vercel 变量改完要 Redeploy。为提升加载速度，我在前端加了懒加载与脚本多源回退。

如果面向国内访问，我会把 Waline 服务端迁到腾讯云并绑定国内域名，或者直接改用 Twikoo（CloudBase 更稳）。

第八步：PV/UV 统计

LeanCloud 在中国区，server_url 用 https://api.lc-cn-n1-shared.com。我在主题里打开页脚统计与文章浏览量，都走 LeanCloud。

总结

这就是我从零到上线的流程：选 Hexo + Fluid，先把写作体验和主题打磨好，再把评论与统计接起来，最后部署与加速。之后就专注内容本身了。
